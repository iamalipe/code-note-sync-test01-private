# The Art of Naming Variables

“There are only two hard things in Computer Science: cache invalidation and naming things.” — _Phil Karlton_

**Camel Case:** fileName, userAccount, themeData

**Pascal Case:** FileName, UserAccount, ThemeData

**Snake Case:** file_name, user_account, theme_data

**Kebab Case:** file-name, user-account, theme-data

**Screaming Case:** FILENAME, USER_NAME

**Hungarian Notation:** Names start with a lowercase prefix to indicate intention.

- **Systems Hungarian:** where prefix indicates data type.  
    Examples: strFirstName, arrUserNames
- **Apps Hungarian:** where prefix indicates logical purpose.  
    Examples: rwPosition, pchName

---

**With Camel Case and Pascal Case:** Pascal, Modula, Java and .NET, etc.

**With Snake Case:** Python, Ruby, C/C++ standard libraries, and WordPress, _**Naming Files**_, etc.

**With Kebab Case:** COBOL, Lisp, Perl 6, and CSS, etc.

---

_**PascalCase**_ for Class Names

_**UPPER_SNAKE_CASE**_ for Class Variables

_**Lower_snake_case**_ for Variables or Global Variables

_**camelCase**_ for Method or Function

---

1. The importance of readability.
2. Prefer explanatory names.
3. Use natural language with argument labels.
4. Avoid pointless function names.
5. Avoid the same name covering many concepts.
6. Avoid the same concept covered by many names.
7. Don’t say one thing and do another.

pluralizing the variable name makes sense.

“**is**”, “**has**”, and **“can**” for Booleans.
